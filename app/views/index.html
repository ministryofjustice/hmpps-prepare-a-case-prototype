Sprint 15

 beta

10 June 2020

Main changes
Change Why we made it
Service name and prototype badge - service name changed to 'Prepare a case for sentence' and prototype badge added Service was renamed to reflect the end to end journey that the service will support.
Court name - court name moved from above H1 and added to primary navigation Users working in multiple courts need to be able to see on all pages which court the case or case list they are looking at relates to.
Case list - H1 change to reflected the day of the case list and caption is 'Cases' A change to navigating days was required as part of the tactical release. The H1 now shows which day of the week the user is looking at.
Day navigation - removed 7 day view and replaced with three day view (today, tomorrow and following day) Removed 7 day view to reflect that only 3 day view would be available in tactical release of the service.
Timestamp - moved to sit in line with case list and added new timestamp of next update Timestamp moved to sit in line with case list tab to reflect the time is related to the case list updating. Next update timestamp added to reflect when the case list was due to update with new cases.
Recently added cases - new tab added to 'Today' caselist to reflect cases added on the day. Users want to be able to see recently added cases in a separate view so they can identify remand/overnight cases or cases that have appeared on the list since it was prepped in advance.
Removed cases - new tab added to 'Today' caselist to show cases that have been removed from the list. To show cases to users that may have been prepped in advance but are no longer appearing on the day's caselist. Included Libra listing number so they can check if case has been relisted at later date.
Possible records banner - added call to action above case list to review possible records To give users an option to complete the reviewing possible records as part of a bulk task.
Possible records badge - change 'Duplicate records' to 'Possible records' Changed to reflect that possible records can be more than just duplicates of the same person.
Review possible records (bulk list) - allow users to see a list of all defendants on a case list with possible record and review records. To allow users to see all defendants on a case list with possible records and review them individually to confirm the correct record and probation status.
Review possible record - allow users to review possible records and confirm the right record for a defendant To allow users to review defendant details and the details of possible records to decide if a record for the defendant exists.

Sprint 14

 beta

27 May 2020

Main changes
Change Why we made it
Requirements - when length of requirement is unknown, use 'unavailable' instead of '-' Change made for accessibility purposes and minimise confusion for users using screenreaders.
Attendance table - change 'Complied' to 'Acceptable' and 'Failures to comply' to 'Unacceptable' Users confusing terms around 'compliance' when reading the table, particularly a 'Failure to comply' absence. Changed to 'Acceptable' and 'Unacceptable' as clearer and users familiar with concept of 'acceptable' or 'unacceptable attendance..
Post release status - add a post release status for persons with a current custodial order User need identified to show if current defendants with a custodial order were on licence or post sentence supervision following their release from custody.
Custodial order summary - change 'Status' on custodial order summary to 'Post release status' Consistency with the language of the status displayed on the previous page.
Appointment attendance - change heading to 'Attendance' Clear to user that attendance can only be in relation to appointments so no need to state in H2.
Case summary - remove details of who sent offender manager report and when it was received Reduce the amount of content on the case summary and content not serving a clear user need on that screen.
Offender manager report - add details of who sent report and when to report screen User needs to know who sent report and when it was received alongside the report.
Probation record - change format to show date of last PSR and how long it has been User needs to know if report is less than 6 months old, return this information as well as the date so user does not need to calculate the age of the report themselves.
Attendance - last attended date made bold Some users were missing the date of the last attended date. Field was made bold to align with the other headings in the attendance section.

Sprint 12

 beta

29 April 2020

Main changes
Change Why we made it
Case details - renamed case summary To allow users to see all the courts available within the service and navigate to them.
Support informaiton - moved CPS pack under supporting information H2 and added in 'offender manager report' status and link to view To allow users to move between different courts available in the service if they work in more than one.
Recent case activity - added timeline component to sidebar of case summary to highlight recent case activity such as appearances and offender manager report requests. To reflect that the service was in a private beta rollout to a small number of test courts.

A deep link through to the contact log in NDelius more appropriate than recreating contact log in our service.
Offender manager report - user has ability to view a report uploaded to the service Added ability to see a completed OM report from the case summary for purpose of user test.

Main changes
Change Why we made it
Key details header - standardised display of Probation status and removed risk of serious harm status Probation status was inconsistent with display on case list. RoSH alone not representative of offender risk picture - needs further research and clarification from policy
Risk registers - removed status column and markers from register table Checked with SME and only RoSH has inherent status levelling. Need further investigation as to why nDelius attributes different colours to certain risk flag types
Personal details - removed details from Probation Not servicing a clear/strong need
Probation record - added details around last PSR and OaSys assessment (includes type and status date) Allows court staff to see if previous PSR can be reused with an addendum if SU is in on a similar offence. Can we reliably link to the document?
Probation record - removed displaying requirements from current orders To ensure design is extensible and can handle multiple concurrent current orders
Order summary - attendance
moved appointment data in to a table view with a more granular breakdown of contact type and non attendance reasons 		We found low confidence in NS summary data and what it represents/how it is derived. A breakdown increased confidence and provides better context to start to assess compliance and engagement at a high level
Order summary - most recent contacts
Added relates to data line in accordion section summary 		Displaying contact type heading alone did not provide enough context

Order summary - custodial order
added supervision status, throughcare dates, licence conditions and pss requirements 		SUs can move through different states of supervision when on a custodial order. Need to call this out and the dates around it

Sprint 6

 beta

5 Feb 2020

Main changes
Change Why we made it
Case list - remove adjournment reason from listing column Adjournment reason being pulled back may not provide any useful information to user e.g 'Adjourned for other reason' or 'Adjourned for non-standard reason'. Removed to reduce content and cognitive load on case list.
Key details banner - remove most recent ROSH score for known defendants Users in court don't prioritise one risk score over another and view risk holistically so usually want to see all risk factors before making a decision.
Person - changed to 'Personal details' To be clearer on the type of information displayed.
Person - defendant details renamed 'Personal details from charge' and 'Offender record details' renamed 'Personal details from Probation Service'. To indicate the source of this information (Libra/Police) and (NDelius).
Order summary - breakdown against each requirement type removed. Amalgamated into requirements and length, as well as overall attendance. Unable to reliably get a breakdown of progress against some requirement types (RAR and accredited programme).
Order summary (contacts) - change format of contacts table into accordion component and add link to full contact log in NDelius. Some contacts can be lengthy. To reduce cognitive load, hide contact note/description behind accordion component with option to show all to users. User testing also indicated a need to see more than most recent contacts.
Risk register - combine Type/Description into single field and pull back notes into service instead of a link out to NDelius to view.
 Remove duplication by combining description and type into single field.

Sprint 5

 beta

22 Jan 2020

Main changes
Change Why we made it
Case list - remove boxes and links to see pre-filtered list of defendants according to probation status. Replace with filter component within case list. Reduce cognitive load on case list screen and allow users to filter the case list as they need to.
Case list - content changes including change sitting to session, remove 'listing' from listing column, indicate when previous order ended for previously known defendants, and remove whether defendant supervised by CRC or NPS. Reduce cognitive load on case list.
Key details - remove appearance information, add in PNC and CRN fields. Moved appearance information to case details as only relevant to the case. PNC and CRN are identifiers which should persist across all defendant views.
Case summary - remove buttons to adjourn or close case, remove OM update, PSR report request, case tracker, pre-cons and notes. Features not required as part of MVP identified for private beta.
Case details navigation - added separate tabs for personal details, probation record and risk registers that can be navigated within a case. Improve navigation between different aspects of case and defendant information.
Probation record - separate current and previous orders, return requirements for current orders, surface OM contact information. Make current and recent order information clearer to user and surface offender manager contact details in order to request an update.
Risk register - display active and inactive risks along with registration dates and associated notes completed.
 Improve visibility and detail of risk information
